1.6.  Zahlen


Zahlen werden standardmäßig in Dezimalschreibweise dargestellt. Reelle (oder Dezimal-)Zahlen werden in technischer oder wissenschaftlicher Notation geschrieben (z. B. 0,314E1).


Für Konstanten vom Typ Integer unterstützt Free Pascal vier Formate:


  1. Normales Dezimalformat (Basis 10). Dies ist das Standardformat.
  2. Hexadezimalformat         (Basis 16), analog zu Turbo Pascal. Um einen konstanten Wert im Hexadezimalformat anzugeben, stellen Sie ihm ein Dollarzeichen ($) voran. Somit entspricht $FF im Hexadezimalformat 255 im Dezimalformat.
    Beachten Sie, dass die Groß- und Kleinschreibung bei hexadezimalen Konstanten keine Rolle spielt.
  3. Ab Version 1.0.7 wird auch das Oktalformat (Basis 8) unterstützt.
    Um eine Konstante im Oktalformat anzugeben, stellen Sie ihr ein Et-Zeichen (&) voran. Beispielsweise wird 15 in Oktalnotation als &17 angegeben.
  4. Binärnotation (Basis 2). Einer Binärzahl kann ein Prozentzeichen (%) vorangestellt werden. Somit kann 255 in Binärnotation als %11111111 angegeben werden.


Die folgenden Diagramme zeigen die Syntax für Zahlen.

digit     ::= [0-9]

hexletter ::= [a-fA-F]

hexdigit  ::= '$' ( digit | hexletter )

oktal ::= '&' digits

bindigit ::= '%' [0-1]*

digit ::= [0-9]*

Vorzeichen ::= '+' | '-'

vorzeichenlose Ganzzahlen ::=

    digit_sequence

  | '$' hexdigit_sequence  /* $CAFE001 */

  | '&' octal_sequence     /* &270     */

  | '%' bindigit_sequence  /* %10100   */

scale_factor ::=

  (E | e)('+'|'-')digit_sequence

unsigned_real ::=

    digit_sequence '.' digit_sequence

  | scale_factor

unsigned_number ::=

    unsigned_real

  | unsigned_integer

signed_number ::=

  ('+' | '-') unsigned_number


Hinweis:

oktal und binäre Notation wird im TP und Delphi kompatibilitätsmodus nicht unterstützt.
